## 남은 작업

### v2 — 송장 엑셀 자동생성

거래명세서 → 택배 송장 Excel도 자동 생성.
현재 수기로 옮기고 있는 작업.

**참고 파일**: `~/Downloads/파일변환예제문서0225.xlsx`
- Sheet 1 (엑셀파일 첫행-제목없음): 실제 사용 중인 형식
  - A: 수하인명 (마켓컬리 창원 / 김포 / 평택)
  - B: 수하인주소
  - D: 전화번호
  - E: 품목명
  - F: 택배수량
  - G: 택배운임
  - H: 운임구분

**구현 방향**:
- `exceljs` 라이브러리로 클라이언트 사이드 Excel 생성
- 거래명세서 파싱 데이터 + 물류센터 정보 → 송장 Excel 다운로드
- 물류센터별 주소/연락처 마스터 데이터 필요 (평택, 김포, 창원 등)

### v3 — 발주 이력 관리

발주를 DB에 저장해서 이력 추적.

**필요한 테이블**:
- `kurly_orders`: id, order_code, delivery_date, manufacture_date, status, created_at
- `kurly_order_items`: id, order_id, product_id, quantity, box_count

**기능**:
- 최근 발주 목록 조회
- 발주별 라벨지 재출력
- 월별 납품 현황 요약

### v4 — 제조 스케줄 연동

발주 입고일 기준으로 제조 일정 자동 계산.

**기능**:
- 입고일 - 리드타임 = 제조 시작일
- 제품별 제조 소요시간 마스터 데이터
- 어드민 캘린더에 제조 일정 표시
- 납품 D-day 알림

### v5 — 추가 개선

- 물류센터 마스터 데이터 관리 (주소, 연락처, 도크 번호)
