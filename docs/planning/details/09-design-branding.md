## 1. 브랜드 아이덴티티

### 1-1. "출첵"의 브랜드 성격

출첵은 **소규모 매장 사장님과 직원** 모두가 매일 쓰는 근태 도구다.
브랜드 성격은 세 가지 키워드로 정의한다.

| 키워드 | 의미 | 브랜드 표현 방식 |
|--------|------|-----------------|
| **신뢰** | 출퇴근 기록은 급여와 직결 — 정확하고 투명해야 한다 | 깔끔한 UI, 수치 명확 표시, 차분한 컬러 |
| **간편** | 20대 알바생도 40대 사장님도 30초 안에 출퇴근 가능 | 최소 탭, 큰 버튼, 직관적 플로우 |
| **활기** | 크레딧/티어 시스템으로 출근이 게이미피케이션된다 | 티어 뱃지, 스트릭 불꽃, 보상 애니메이션 |

**브랜드 보이스**: 토스 UX 라이팅 원칙 그대로 유지 (`~해요` 체, 친근, 공감)

**브랜드 슬로건 후보**:
- "출근도 스마트하게" (기능 중심)
- "매일의 출근이 쌓이는 곳" (크레딧/성장 중심)
- "우리 매장 근태, 한눈에" (사장님 관점)

### 1-2. 네이밍 검증

#### "출첵" 이름 분석

### 1-3. 대안 이름 후보

"출첵" 상표 문제 발생 시 백업 후보:

| 이름 | 장점 | 단점 |
|------|------|------|
| **체크인** (Checkin) | 직관적, 글로벌 확장 가능 | 너무 일반적, 동명 서비스 다수 |
| **출근부** (Chulgeunbu) | "출근부"라는 익숙한 개념 | 올드한 느낌 |
| **타임첵** (TimeChek) | 시간+체크, 영문 친화 | 영어 혼합으로 한글 기억도 떨어짐 |
| **출출** (ChulChul) | 귀엽고 기억 쉬움, "출근+출석" | 음식 연상("출출하다") |
| **근태리** (Geuntaeri) | "근태+관리" 축약 | 사장님에게는 OK, 직원에겐 딱딱 |

> 현 시점에서는 "출첵"이 가장 적합. 상표 검색 결과에 따라 최종 확정.

## 2. 로고 디자인 방향

### 2-1. 로고 구성

```
[심볼] + [워드마크] 조합형

1) 심볼 단독 사용: 앱 아이콘, 파비콘, 작은 UI 요소
2) 심볼 + 워드마크: 로그인 화면, 랜딩 페이지, 명함
3) 워드마크 단독: 텍스트만 필요한 곳 (크레딧 카드 하단 등)
```

### 2-2. 컨셉 키워드

로고에 녹여야 할 핵심 메타포:

| 키워드 | 시각적 메타포 |
|--------|-------------|
| 출석 | 체크 마크(V), 도장, 스탬프 |
| 시간 | 시계, 타임라인 |
| 위치 | 핀 마커, GPS 동심원 |
| 연결 | 사람-매장 연결, 네트워크 |

**추천 심볼 방향**: 체크 마크(V)를 위치 핀과 결합한 형태
- 위치 핀의 하단 뾰족한 부분이 체크 마크로 변형
- 또는 체크 마크 안에 시계 요소 배치
- 라운드/소프트한 형태 (카페/식당 친화적)

### 2-3. 참고 스타일 (한국 SaaS)

| 서비스 | 로고 특징 | 참고 포인트 |
|--------|----------|------------|
| **토스** | 파란 로고, 미니멀, 심볼(새) + 워드마크 | 신뢰감, 깔끔함 |
| **당근** | 주황 당근, 친근하고 따뜻한 느낌 | 아이콘성 심볼, 따뜻한 컬러 |
| **배달의민족** | 블랙 + 민트, 독특한 한글 타이포 | 한글 중심 브랜딩 |
| **시프티** (Shiftee) | 근태관리 SaaS, 보라/파랑, 미니멀 | 같은 카테고리 경쟁 참고 |
| **알밤** | 출퇴근 앱, 파란색 계열, 체크 마크 | 직접 경쟁, 차별화 필요 |
| **샤플** (Shopl) | 매장관리 SaaS, 파란색, 미니멀 | 같은 타겟 사장님 |

**차별화 포인트**: 크레딧/게이미피케이션 요소 (다른 근태 앱에 없는 고유 특징)

### 2-4. 로고 제작 방법

| 방법 | 비용 | 품질 | 소요 기간 | 추천도 |
|------|------|------|----------|--------|
| **A. 직접 제작** (Figma) | 무료 | 본인 역량 의존 | 1~3일 | MVP 단계에서 가능 |
| **B. AI 도구 활용** | 무료~2만원 | 중 | 1일 | 초안 + 수정에 적합 |
| **C. 크라우드소싱** (라우드소싱) | 10~30만원 | 중상 | 7~14일 | 가성비 좋음 |
| **D. 전문 디자이너** (숨고/크몽) | 20~80만원 | 상 | 7~21일 | 정식 런칭 시 추천 |
| **E. 디자인 에이전시** | 200만원+ | 최상 | 1~2개월 | 시리즈 A 이후 |

**MVP 추천 전략**:
1. AI 도구(Midjourney, DALL-E, Looka)로 컨셉 시안 5~10개 생성
2. Figma에서 가장 좋은 시안 기반으로 벡터화 + 정리
3. 정식 출시 전에 크몽/숨고에서 전문 디자이너에게 리파인

**AI 로고 생성 도구**:
- **Looka** (looka.com): 텍스트 입력으로 로고 자동 생성, 한글 지원
- **Brandmark** (brandmark.io): AI 기반 로고+브랜드 가이드 생성
- **Midjourney**: 프롬프트로 심볼 컨셉 탐색 후 Figma에서 트레이싱

## 3. 컬러 시스템

### 3-1. 기존 토스 블루 유지 vs 독자 컬러

| 옵션 | 장점 | 단점 |
|------|------|------|
| **A. 토스 블루 유지 (#3182F6)** | 이미 구현된 코드 전체 재활용, 신뢰감 검증됨 | 토스 모방 느낌, 독자성 부족 |
| **B. 독자 Primary 컬러** | 브랜드 차별화, 고유 정체성 | 전체 UI 재작업 필요 |
| **C. 토스 블루 미세 조정** | 최소 코드 변경, 미묘한 차별화 | 큰 차별화 효과 적음 |

**결정: 옵션 A (토스 블루 유지) + 보조 컬러로 차별화**

이유:
- 현재 90개 이상의 파일에서 `#3182F6` 사용 중 — 전면 변경 시 리스크 큼
- 파란색은 신뢰/안정의 범용 컬러 — 토스만의 소유물이 아님
- 시프티, 알밤 등 근태 앱도 파란색 계열 사용 (카테고리 컬러)
- **Secondary/Accent 컬러로 차별화** + 로고/아이콘 디자인으로 정체성 확보

### 3-2. 컬러 팔레트

| `primary-dark` | `#1B64DA` | Hover/Active 상태 (신규) |

민트/틸 계열 선택 이유:

- 카페/식당의 신선하고 활기찬 이미지
- 토스(파랑 단색)와 차별화 — 출첵은 "파랑+민트" 듀오톤

```
  - 체크 마크 좌측 하단에 미세한 위치핀 힌트 (선택)

대안 심볼:
  - 시계 + 체크 마크 결합
  - 캘린더 아이콘 + 체크 마크
  - "출" 한글 레터링 (배민 스타일 한글 강조)
```

## 5. 크레딧 카드 디자인

### 5-1. 개요

블루프린트(섹션 4-2, 4-3)에서 정의한 크레딧 카드를 **모달 UI**로 구현한다.
`CreditCardModal` 컴포넌트에서 카드를 렌더링하고, `html2canvas`로 이미지 캡처 후 공유한다.

### 5-2. 모달 UI 레이아웃

```
┌─────────────────── CreditCardModal (바텀시트) ───────────────────┐
│                                                                    │
│  [X] 닫기                                                         │
│                                                                    │
│  ┌─────────── 카드 영역 (캡처 대상 #credit-card) ───────────┐   │
│  │                                                              │   │
│  │  [출첵 로고 워드마크]                                       │   │
│  │                                                              │   │
│  │  ┌──────┐  홍길동                                          │   │
│  │  │아바타│  연경당 카페 외 1곳                               │   │
```

### 5-3. 티어별 비주얼 디자인

각 티어는 고유한 시각적 테마를 가진다. 크레딧 카드의 배경/장식이 티어에 따라 변한다.

| 티어 | 배경색 | 장식 패턴 | 배지 색 | 텍스트 색 |
|------|--------|----------|---------|----------|
| **다이아몬드** | 그라디언트: `#E8F0FE` → `#D0E4FC` | 미세한 다이아몬드 격자 패턴 (SVG) | `#B8D4F5` | `#1A4A8A` |
| **플래티넘** | 그라디언트: `#EAECEF` → `#D4D8DC` | 수평 라인 패턴 (빗금) | `#C8D4DC` | `#4A5568` |
| **골드** | 그라디언트: `#FDF6E3` → `#F5E6B8` | 미세한 별/반짝임 SVG | `#C9A84C` | `#7B6320` |
| **실버** | 그라디언트: `#F0F2F5` → `#D8DCE0` | 원형 도트 패턴 | `#9BAAB8` | `#4A5568` |

| **아이언** | `#F2F4F6` (단색) | 없음 (플레인) | `#78828C` | `#4A5568` |

#### 티어별 특수 효과 (선택)

- **다이아몬드**: 카드 상단에 미세한 반짝임(shimmer) CSS 애니메이션
- **골드**: 배경에 은은한 광택(shine) 효과
- **나머지**: 정적 디자인

### 5-4. 공유 시 이미지 사이즈

#### 카카오톡 최적화

| 항목 | 값 | 비고 |
|------|-----|------|
| 권장 이미지 비율 | 1:1 또는 2:3 | 카카오 피드 메시지 기준 |
| 카드 캡처 해상도 | 800 x 1200 px | @2x 캡처 후 리사이즈 |
| 카카오 피드 이미지 | 800 x 800 px (1:1) | `imageUrl`에 사용 |
| 파일 형식 | PNG | 투명도 불필요하면 JPEG도 가능 |
| 최대 파일 크기 | 5MB (카카오 제한) | 보통 500KB 이하 |

#### html2canvas 캡처 사이즈 계산

```typescript
// 카드 DOM 요소의 실제 크기를 @2x로 캡처
const canvas = await html2canvas(cardElement, {
  scale: 2,                    // Retina 대응
  useCORS: true,               // 외부 이미지(아바타) 허용
  backgroundColor: '#FFFFFF',  // 배경색 명시
  width: 400,                  // 카드 실제 너비 (px)
  height: 600,                 // 카드 실제 높이 (px)
});
// 결과: 800 x 1200 캔버스
```

### 5-5. html2canvas 렌더링 고려사항

| 주의사항 | 해결 방법 |
|---------|----------|
| **외부 이미지 CORS** | 아바타 이미지는 Supabase Storage 서명 URL → `useCORS: true` |
| **커스텀 폰트 미렌더링** | `html2canvas`가 @font-face를 인식 못할 수 있음 → 폰트를 Base64로 인라인 or `font-display: swap` 사용 |
| **SVG 렌더링** | `html2canvas`는 일부 SVG를 무시 → 티어 뱃지 SVG는 `foreignObject` 사용 지양, 이미지로 대체 |
| **그라디언트 배경** | CSS gradient는 정상 캡처됨 |
| **Tailwind 클래스** | 캡처 대상 요소에 인라인 `style` 우선 적용 권장 |
| **Emoji 렌더링** | OS별 이모지 차이 → 티어 이모지를 SVG 아이콘으로 대체 검토 |
| **모바일 Safari** | `html2canvas` 성능 이슈 → `scale: 2` 이상 지양 |

#### html2canvas 대안 비교

| 라이브러리 | 장점 | 단점 |
|-----------|------|------|
| **html2canvas** | 가장 범용, 레퍼런스 많음 | SVG 일부 미지원, 느림 |
| **dom-to-image** | 더 정확한 SVG 렌더링 | 유지보수 중단 우려 |
| **modern-screenshot** | 최신, ESM, 빠름 | 레퍼런스 적음 |

> 추천: `html2canvas` 우선 사용. SVG 이슈 발생 시 `modern-screenshot`으로 전환.

### 8-2. 크레딧 카드용 특수 폰트 (선택)

크레딧 카드의 숫자/점수 표시에 **디스플레이용 숫자 폰트**를 사용하면 프리미엄 느낌을 줄 수 있다.

| 폰트 | 특징 | 라이선스 | 추천도 |
|------|------|---------|--------|
| **Space Grotesk** | 기하학적, 모던, 숫자가 예쁨 | SIL OFL (무료) | 높음 |
| **Inter** | 범용성, 숫자 가독성 | SIL OFL (무료) | 중 |
| **DM Sans** | 소프트하고 둥근, 카페 느낌 | SIL OFL (무료) | 중 |
| **Outfit** | 둥글고 부드러운 디스플레이 | SIL OFL (무료) | 높음 |

**추천**: Space Grotesk 또는 Outfit을 크레딧 카드의 **점수 숫자**에만 적용.

```typescript
// 크레딧 카드에서만 사용

  920
</p>
```

> 주의: html2canvas 캡처 시 폰트 로딩 필수 — `document.fonts.ready` 대기 후 캡처.

## 10. 디자인 시스템 정리

### 10-1. Tailwind 토큰 업데이트

현재 `globals.css`에서 shadcn/ui 기본 토큰을 사용 중.
출첵 고유 토큰을 추가한다.

#### globals.css 추가 토큰

```css
:root {
  /* 기존 토큰 유지 */

  /* 출첵 고유 토큰 (신규) */
  --chulchek-primary: #3182F6;
  --chulchek-primary-light: #E8F3FF;
  --chulchek-primary-dark: #1B64DA;
  --chulchek-secondary: #00C9A7;
  --chulchek-secondary-light: #E6FBF6;
  --chulchek-secondary-dark: #00A88C;
  --chulchek-accent-warm: #FF8C42;
  --chulchek-accent-warm-light: #FFF4E6;

  --chulchek-text-primary: #191F28;
  --chulchek-text-secondary: #4E5968;
  --chulchek-text-tertiary: #8B95A1;
  --chulchek-text-disabled: #B0B8C1;

  --chulchek-bg-default: #F2F4F6;
  --chulchek-border: #E5E8EB;

  --chulchek-success: #20C997;
  --chulchek-warning: #D9480F;
  --chulchek-error: #E03131;
}
```

#### @theme inline 확장

```css
@theme inline {
  /* 기존 토큰 유지 */

  /* 출첵 시맨틱 토큰 (신규) */
  --color-chulchek-primary: var(--chulchek-primary);
  --color-chulchek-primary-light: var(--chulchek-primary-light);
  --color-chulchek-secondary: var(--chulchek-secondary);
  --color-chulchek-secondary-light: var(--chulchek-secondary-light);
  --color-chulchek-accent-warm: var(--chulchek-accent-warm);
}
```

> 이렇게 하면 `text-chulchek-primary`, `bg-chulchek-secondary-light` 등 Tailwind 유틸리티로 사용 가능.

### 10-2. shadcn/ui 커스텀 테마

현재 shadcn/ui 기본 neutral 테마 사용 중.
출첵 컬러로 매핑 업데이트가 필요한 토큰:

| shadcn 토큰 | 현재 값 | 출첵 값 | 비고 |
|------------|--------|--------|------|
| `--primary` | oklch(0.205 0 0) (검정) | `#3182F6` 변환 | CTA 버튼 등에 사용 |
| `--primary-foreground` | oklch(0.985 0 0) (흰) | `#FFFFFF` 유지 | |
| `--destructive` | oklch(0.577 ...) | `#E03131` 변환 | |
| `--ring` | oklch(0.708 0 0) | `#3182F6` 변환 | 포커스 링 |

다만 현재 코드에서 shadcn 토큰보다 **인라인 색상 (`text-[#3182F6]`, `bg-[#E8F3FF]`)**을 직접 사용하는 경우가 대부분이므로, shadcn 토큰 변경 영향은 제한적.

**장기 계획**: 인라인 색상 → CSS 변수/Tailwind 토큰으로 점진적 마이그레이션.
