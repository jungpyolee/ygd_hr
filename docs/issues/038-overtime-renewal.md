# 038 — 추가근무 관리 전면 리뉴얼

## 배경

기존 추가근무 관리 페이지(`/admin/overtime`)는 타임라인 시각화 중심의 복잡한 UI로 설계됐으나,
실제 카페 사장 입장에서의 워크플로우와 맞지 않아 전면 재설계.

---

## 핵심 컨셉

> "추가근무는 직원이 주장하는 게 아니라, 사장이 인정해주는 것"

- 사장 중심 단방향 앱
- 직원 요청 기능 없음 (DB 구조의 `pending` 상태도 미사용)
- 사장이 직원 출퇴근 현황을 보고 직접 판단 → 빠르게 등록

---

## 기존 문제점

| 문제 | 원인 |
|------|------|
| 이중 레이아웃 (모바일) | 페이지가 `min-h-screen` + 자체 `<header sticky>` 추가 → AdminLayout과 중복 |
| 타임라인 시각화 복잡 | 사장이 원하는 건 "몇 분 초과?" 하나 |
| "최근 14일" 고정 | 날짜 선택 불가 |
| +1분~+1시간 버튼 고정 | 업체마다 단위가 다름 |
| 직접 할당 vs 빠른 승인 이원화 | 흐름 분리로 혼란 |
| 확인 완료 개념 없음 | 처리됐는지 알 수 없음 |
| `start_time` / `end_time` 저장 | 시간 크기(분)만 필요한데 시간대 저장 |

---

## 새로운 설계

### 1. 레이아웃 수정

다른 어드민 페이지(`employees`, `announcements` 등)와 동일하게:
- `min-h-screen` 래퍼 제거
- 페이지 자체 `<header sticky>` 제거
- AdminLayout의 `<main>` 안에 콘텐츠만 렌더링
- 페이지 타이틀은 일반 `<h1>` 또는 상단 섹션으로 처리

### 2. DB 변경

#### `overtime_requests` 테이블 변경

```sql
-- 제거
start_time  time
end_time    time

-- 추가
minutes     integer NOT NULL  -- 인정된 추가근무 분

-- status enum 변경
-- 기존: 'pending' | 'approved' | 'rejected'
-- 변경: 'approved' | 'dismissed'
-- (pending은 직원 요청 기능 미사용으로 제거)
```

#### `stores` 테이블 변경 (매장 설정)

```sql
-- 추가
overtime_unit          integer DEFAULT 30        -- 인정 단위 (분): 15, 30, 60, 0(자유)
overtime_include_early boolean DEFAULT false     -- 일찍 출근 포함 여부
overtime_min_minutes   integer DEFAULT 10        -- 최소 인정 기준 (분)
```

### 3. 상태 체계

직원/날짜 조합마다 3가지 상태:

| 상태 | 의미 | DB 저장 |
|------|------|---------|
| `pending` | 사장이 아직 확인 안 함 | overtime_requests 레코드 없음 |
| `approved` | 추가근무 인정 | status = 'approved' |
| `dismissed` | 확인했고 넘김 | status = 'dismissed' |

날짜 레벨: 해당 날짜 직원 전원이 `approved` 또는 `dismissed` → **확인 완료**

### 4. 직원 케이스 분류

| 케이스 | 조건 | 표시 |
|--------|------|------|
| A: 스케줄 초과 | 스케줄 있음 + 실제 근무 > 스케줄 | "스케줄 초과" 섹션 |
| B: 스케줄 없음 | 스케줄 없음 + 출퇴근 기록 있음 | "스케줄 없음" 섹션 |
| C: 정상 범위 | 초과 시간 < 최소 기준 | 표시 안 함 (확인 필요 아님) |

## UI 구조

### 메인 화면 (날짜 목록)

```
추가근무 관리

── 확인 필요 ─────────────────
  3월 24일 (오늘)   3명 대기   >
  3월 23일          1명 대기   >

── 확인 완료 ─────────────────
  3월 22일          추가근무 2건  >
  3월 21일          전원 넘김     >
```

- 확인 필요 섹션이 상단 고정
- 처리 완료 시 자동으로 확인 완료로 이동
- 어드민 대시보드 카드에 미처리 건수 뱃지 표시

### 날짜 상세 화면

```
3월 23일

── 확인 필요 ─────────────────────────────
┌──────────────────────────────────────┐
│ 👤 이준혁   스케줄 초과               │
│ 스케줄  14:00~22:00                   │
│ 실제    14:05~22:40                   │
│ 늦게 퇴근  +40분                      │
│                                      │
│ [넘기기]  [30분] [1시간] [직접입력]   │
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│ 👤 박서연   스케줄 없음               │
│ 실제  10:05~18:12 (8시간 7분)         │
│ 이 날 스케줄이 없어요.                │
│                                      │
│ [넘기기]  [전체 등록]  [직접입력]     │
└──────────────────────────────────────┘

── 추가근무 인정 ──────────────────────────
┌──────────────────────────────────────┐
│ 👤 김민지   30분 인정됨               │
│                              [취소]  │
└──────────────────────────────────────┘

── 넘김 ──────────────────────────────────
┌──────────────────────────────────────┐
│ 👤 최유진   늦게 퇴근 +12분 → 넘김    │
│                        [추가근무로]  │
└──────────────────────────────────────┘
```

### 직접 입력 바텀시트

매장 단위 설정이 30분이면 → 30분 단위 피커
자유 입력이면 → 시간/분 스크롤 피커

```
┌─────────────────────────────┐
│ 추가근무 시간               │
│                             │
│    1  시간  30  분          │
│   ────────  ────            │
│   스크롤    스크롤          │
│                             │
│ 등록하기                    │
└─────────────────────────────┘
```

시작 시간은 스케줄 종료 시각 기준 자동 세팅. 사장은 "몇 분"만 결정.

### 매장 설정 화면 (기존 설정 페이지에 섹션 추가)

```
추가근무 규칙

인정 단위
  ○ 15분   ● 30분   ○ 1시간   ○ 자유 입력

인정 기준
  ☑ 늦게 퇴근한 시간
  ☐ 일찍 출근한 시간

최소 인정 기준
  [  10  ] 분 이상일 때만 표시
```

설정에 따라 날짜 상세의 빠른 버튼이 자동 변경.

| 이벤트 | 알림 내용 |

- 넘김(dismissed) 처리 시에는 알림 없음
