# 049 — 컬리 발주 라벨지 자동생성

## 배경

연경당은 HACCP 인증 공장에서 마켓컬리에 제품을 납품하고 있다.
기존 발주 처리 프로세스:

1. 컬리 셀러 포털에서 **거래명세서 PDF** 다운로드
2. PDF를 보면서 **입고라벨지 Excel**에 제품/수량/박스 정보를 수기 작성
3. **송장 Excel**에도 동일한 정보를 수기 작성
4. 각각 인쇄 후 박스에 부착

제품별 박스별로 라벨을 하나하나 채워야 해서 시간이 많이 소요됨.
발주 1건당 평균 5~7개 라벨을 수작업으로 작성하는 상황.

## 해결

거래명세서 PDF 업로드 → 자동 파싱 → 라벨지 자동생성 → 브라우저 인쇄.
수작업을 **PDF 올리고 인쇄 누르면 끝**으로 줄임.

## 구현 내용

### 1. DB — kurly_products 테이블 (마이그레이션 043)

제품 마스터 테이블. 거래명세서의 마스터코드와 매칭하는 용도.

| 컬럼 | 설명 |
|------|------|
| name | 상품명 (예: [연경당] 금귤정과 140g) |
| master_code | 컬리 마스터코드 (UNIQUE) |
| barcode | 바코드 |
| unit_weight | 규격 중량 |
| box_capacity | 박스당 입수량 |
| is_active | 활성 여부 |

초기 등록 제품 8종:
- 금귤정과 140g (M00000420174, 28개입)
- 쇼핑백 증정품 (M00000396556, 18개입)
- 견과류 강정 2구 세트 (M00000255955, 18개입)
- 메밀 강정 125g (M00000255954, 28개입)
- 참깨 강정 160g (M00000255953, 28개입)
- 피칸 강정 130g (M00000255952, 28개입)
- 호두 강정 120g (M00000255951, 28개입)
- 곶감단지 선물세트 (M00000386275, 8개입)

### 2. PDF 파싱 API — `/api/orders/parse-pdf`

거래명세서 PDF에서 자동 추출하는 데이터:
- 발주코드 (예: T20260326_4YDOE)
- 입고일 (예: 2026-03-26)
- 제조일자 (예: 2026-03-21)
- 제품별: 마스터코드, 총수량, 박스수, 박스당입수량

파싱 라이브러리: `pdf-parse` v5 (PDFParse 클래스)
- `next.config.ts`에 `serverExternalPackages: ["pdf-parse"]` 추가 필요

파싱 검증 체계:
- 발주코드 없음 → 에러 (컬리 거래명세서 아님)
- 입고일 없음 → 에러
- "연경당" 키워드 없음 → 에러
- 마스터코드 있는데 수량/날짜 파싱 실패 → 에러 (양식 변경 감지)
- 제품 0개 파싱 → 에러
- 박스수 × 입수량 ≠ 총수량 → 경고 (warnings)
- 비정상 수치 → 경고

PDF 텍스트 파싱 주의사항:
- `\x00` null 문자 → 공백으로 치환
- 긴 상품명이 2줄로 나뉨 → 마스터코드 라인 뒤에 날짜 없으면 다음 줄 합치기
- 순번과 마스터코드 사이 공백 유무: `1M000...` 또는 `1 M000...` 둘 다 처리

### 3. 발주 라벨지 페이지 — `/admin/orders`

주요 흐름:
1. 거래명세서 PDF 업로드 (드래그앤드롭 / 클릭)
2. 발주코드, 입고일, 제조일자, 제품+수량 자동 채우기
3. 유통기한은 선택 입력 (제조일자만으로 인쇄 가능)
4. 라벨 미리보기 + "인쇄하기" 버튼
5. 새 창에서 인쇄 최적화 HTML 렌더링 → window.print()

라벨 양식 (컬리 지정):
- A4 1장에 4개 라벨 (2x2 배치)
- 각 라벨: 발주코드, 공급사명, 상품명, 상품코드, 유통기한/제조일자, 수량/총수량, C/T

수동 입력도 가능 (PDF 없이 직접 제품 선택 + 수량 입력).

### 4. 제품 관리 페이지 — `/admin/orders/products`

컬리 제품 마스터 CRUD. 새 제품이 나오면 여기서 등록.
거래명세서 업로드 시 미등록 마스터코드는 에러 토스트로 알려줌.

### 5. 어드민 메뉴

`/admin/layout.tsx`의 menus 배열에 "발주 관리" 항목 추가 (Package 아이콘).
