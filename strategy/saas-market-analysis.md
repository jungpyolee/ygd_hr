## 3. 세무 처리 방향

직접 구현보다 **제휴 방식** 추천.

- 근태 + 급여 계산은 직접 구현
- 세무 신고는 삼쩜삼, 택스워크 등과 제휴 → 중개 수수료 수익
- "세무처리 대행" 표현은 세무사법 위반 소지 → 금지

## 4. 1달 개발 로드맵

### 현재 있는 것
- 위치 기반 출퇴근
- 관리자 대시보드
- 실시간 알림
- 다중 매장 지원
- PWA
- 직원 프로필 관리

### 없는 것 (SaaS 필수)
- 사장님 자체 회원가입/온보딩
- 직원 초대 시스템 (초대링크/코드)
- 요금제 + 결제 (구독)
- 급여 계산 (시급 × 근무시간)
- 급여 명세서 생성/발송
- 근무 스케줄 등록

### 주차별 계획

| 주차 | 작업 |
|------|------|
| 1주차 | 멀티테넌트 구조 전환, 사장님 회원가입, 직원 초대 링크 |
| 2주차 | 급여 계산 (시급 × 근무시간, 주휴수당) |
| 3주차 | 급여 명세서 PDF, 스케줄 등록 |
| 4주차 | 토스페이먼츠 결제 연동, 랜딩페이지 |

### 추천 전략
결제 빼고 D+30 베타 무료 출시 → 실사용자 10~20명 확보 → D+45 유료 전환

## 5. 도입 저항 분석

### 사장님 저항 (가장 큰 허들)

| 유형 | 내용 |
|------|------|
| 돈 저항 | "카톡으로 사진 찍으면 되는데 왜 돈 내?" |
| 기술 저항 | "나 이런 거 잘 몰라", 40~50대 비율 높음 |
| 신뢰 저항 | 개인정보 우려, 스타트업 불신 |
| 필요성 저항 | "알바 3명인데 굳이?", 문제를 문제로 인식 못 함 |

**전환 트리거** — 한 번 당하면 저항 제로:
- 알바 출퇴근 시간 조작 경험
- 주휴수당 계산 틀려서 노동부 신고 당한 경험
- 야간 알바 실제 출근 여부 확인 불가 경험

### 직원(알바) 저항 (상대적으로 낮음)

| 유형 | 내용 |
|------|------|
| 위치 추적 거부감 | "왜 내 위치를 앱이 알아야 해?" |
| 감시 느낌 | "사장이 실시간으로 나 보고 있는 거잖아" |
| 귀찮음 | "어차피 몇 달 일하고 말 건데" |

**저항이 낮은 이유:**
- 고용관계 특성상 사장이 "써야 한다"면 쓰는 구조
- 이미 CCTV 하에서 일하는 게 기본값 → 위치 기반 출퇴근이 상대적으로 덜 거슬림
- 이직률 높아 저항해도 금방 교체

→ **직원 저항보다 사장님 도입 저항이 훨씬 큰 허들**

---

## 6. 마케팅 방향

경쟁사 전략:
- 시프티: 스케줄 관리로 진입 → 근태로 확장
- 알밤: 완전 무료 시작 → 규모 커지면 유료
- 포스기 회사들: POS 끼워팔기

**이 섹터 저항을 뚫는 공식:**
```
무료 + 쉬움 + "이런 거 당해봤죠?" 공감 마케팅
```

핵심 카피: **"알바 출퇴근 조작 막는 앱"**
채널: 카카오 소상공인 카페, 알바몬 커뮤니티, 네이버 블로그
