# 긴급 출근 매칭 기능

## 개요

요식업에서 알바 펑크는 월 1-2회 기본 발생. 현재 사장들은 단톡방에 "급하게 나와줄 사람?" 올리는 구조.
이걸 앱 안에서 해결하면 사장이 앱을 끊을 수 없는 이유가 생김.

---

## 핵심 플로우

```
직원: "나 지금 나갈 수 있어" → 가용 상태 ON (시간 설정)
사장: 펑크 발생 → 가용 직원 목록 확인 → 오퍼 발송
직원: 푸시 알림 수신 → 수락/거절
사장: 수락 확인 → 완료
```

---

## 직원 사이드

### 가용 상태 설정
- ON/OFF 토글
- ON 시 활성화 시간 선택: 1시간 / 2시간 / 3시간 / 직접 입력
- 시간 만료 시 자동 비활성화

### 오퍼 수신
- 푸시 알림 → 앱 진입 → 바텀시트
  - 매장명 + 거리
  - 요청 출근 시간
  - 예상 근무 시간
  - 메모 (사장 입력값)
  - [수락하기] / [거절하기]
- 오퍼 만료 시간 표시 (10분 내 응답)

---

## 사장 사이드

### 가용 직원 목록
- 현재 출근 가능한 직원 카드 리스트
- 카드 표시 항목: 이름 / 크레딧 점수 + 티어 / 가용 잔여 시간 / 마지막 출근일
- 크레딧 높은 순 기본 정렬 (강제 필터 아님 — 사장 재량)

### 오퍼 발송
- 출근 요청 시간 선택
- 근무 시간 입력 (선택)
- 메모 입력 (선택)
- 수락 시 푸시 알림 수신

---

## 크레딧을 참고 지표로만 쓰는 이유

- 강제 기준으로 쓰면 신규 직원 불이익
- 크레딧 낮아도 단골 알바면 사장이 오퍼 가능해야 함
- 정렬 기준은 제공하되 필터는 사장 재량

---

## 중복 오퍼 처리

- 여러 사장이 같은 직원에게 오퍼 가능
- 직원이 수락하면 나머지 오퍼는 자동 만료
- 해당 사장들에게 "이미 다른 곳 출근 확정" 알림

---

## 자동 만료

- 직원 가용 상태: 설정 시간 후 자동 OFF
- 오퍼: 발송 후 10분 내 미응답 시 자동 expired
- Vercel Cron 활용 (이미 daily-settlement에 존재)

---

## DB 설계 (안)

```sql
-- 직원 가용 상태
availability_slots
  id, user_id, store_id
  active_until        -- 만료 시각
  created_at

-- 긴급 오퍼
emergency_offers
  id, store_id, employee_id, sent_by
  requested_start_time
  message
  status              -- pending / accepted / declined / expired
  expires_at          -- 오퍼 만료 (10분)
  responded_at
  created_at
```

## 기존 인프라 활용

| 기존 코드 | 활용 방식 |
|----------|----------|
| 푸시 알림 시스템 | 오퍼 수신 알림 |
| 크레딧/티어 | 직원 카드 배지 표시 |
| StoreDistanceList | 매장-직원 거리 계산 |
| Vercel Cron | 가용 상태 / 오퍼 자동 만료 |
| Realtime | 수락 즉시 사장 화면 업데이트 |
