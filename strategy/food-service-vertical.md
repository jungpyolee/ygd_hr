# 요식업 버티컬 특화 전략

## 핵심 포지셔닝

> 기능 추가가 아닌 "어떤 고통을 없애주느냐"가 차별화.
> 급여 자동 계산 하나만 제대로 돼도 사장들이 매달 2-3시간 쓰는 엑셀 작업이 사라짐.
> 그게 월 1-2만원 구독료의 근거.

---

## 차별화 포인트 (타 서비스 대비)

| 포인트 | 설명 |
|--------|------|
| 노무 리스크 방어 | 근태 기록하면서 법적 리스크까지 해석해주는 앱 |
| 급여 자동화 | 근태 → 급여 명세서 원스텝 |
| 직원 투명성 | 직원이 자기 기록/급여 실시간 확인 → 직원이 사장에게 앱 추천하는 역전 구조 |
| 요식업 버티컬 | 시프티 등은 너무 general — 요식업 특화로 집중 |

---

## 추가 필요 기능 (우선순위순)

### Phase 1 — 구독료 받을 수 있는 수준 (3개월)

#### 1. 급여 자동 계산 + 명세서
- 근태 데이터 기반 자동 계산 항목
  - 기본급 (시급 × 시간)
  - 주휴수당 (주 15시간 이상 시 자동)
  - 야간수당 (22시-06시 × 1.5)
  - 연장수당 (8시간 초과 × 1.5)
  - 식대 (설정값)
- 직원별 급여 명세서 PDF 생성
- 카카오톡/문자 발송

#### 2. 브레이크타임 분할 근무
- 요식업 특유의 점심/저녁 split shift 처리
- 중간 공백은 근무시간 미포함
- 급여 계산 시 두 타임 합산

#### 3. 노무 리스크 자동 감지
- 주 52시간 초과 예정 직원 경보
- 4시간 근무 시 30분 휴게 미부여 감지
- 최저임금 미달 위험 (시급 설정 기반)
- 주휴수당 발생 조건 충족 알림

---

### Phase 2 — 요식업 전용 포지셔닝 (6개월)

#### 4. 포지션 기반 스케줄
- 홀/주방/배달/카운터 포지션별 필요 인원 설정
- 포지션 공백 자동 감지 ("토요일 주방 1명 부족")
- 현재 스케줄 시스템에 포지션 레이어 추가

#### 5. 인건비율 대시보드
- 매출 직접 입력 (or 배달앱 연동)
- 인건비 / 매출 비율 자동 계산
- 업종 평균 대비 경보 (요식업 적정선 25-30%)

#### 6. 위생/법정 관리 강화
- 건강증명서 만료일 D-30/D-7 알림 (현재 health_cert_warning_days 있음 — 확장)
- 위생교육 이수일 관리
- 근로계약서 만료 알림
- 노동청/식약처 점검 대비 즉시 증빙 출력

---

### Phase 3 — 락인 (1년)

| 기능 | 필요 연동 | 난이도 |
|------|----------|--------|
| 배달앱 매출 자동 연동 | 배민/쿠팡이츠 API | 높음 |
| POS 연동 | 포스피아/토스페이먼츠 | 높음 |
| 세금계산서 연동 | 홈택스 | 매우 높음 |

> 연동은 나중에 — 지금은 수동 입력으로 충분.

## AI/AX 접목 시 주의사항

- 소규모 요식업 사장은 AI 기능 자체보다 "고통 제거"에 반응
- 기능은 있으면 좋지만 "돈 내는 이유"가 되지 않음
- AI는 기본 기능 완성 후 차별화 레이어로 얹는 것이 맞음
- 월 AI API 비용은 매장당 $1 미만으로 비용 문제 없음
